# 春原營造官網改版 樣板型產品規格書

**文件版本**：v1.0  
**更新日期**：2025-12-18  
**專案類型**：企業官網樣板開發  
**技術架構**：Vite + Tailwind CSS + 原生 JavaScript

---

## 1. 專案概述

### 1.1 專案背景

本專案為春原營造企業官網改版，採用「樣板型產品」開發模式，提供完整的資訊架構、設計系統、互動邏輯與 RWD 框架，但所有內容皆為占位資料。客戶需依照本規格書提供之「內容提供清單」填寫實際內容，由專案團隊進行最終套版與上線。

### 1.2 目標使用者

- **主要使用者**：外部訪客（潛在客戶、合作夥伴、求職者）
- **次要使用者**：企業內部員工（透過獨立入口進入員工專區）

### 1.3 核心目標

1. 建立專業、現代化的企業形象
2. 有效展示工程實績與核心能力
3. 提供清晰的聯絡與合作管道
4. 支援多裝置瀏覽體驗（Desktop / Tablet / Mobile）
5. 便於長期維護與內容更新

## 2. 樣板交付範圍

本專案交付內容為**完整可運行的網站樣板**，包含以下範圍：

### 2.1 頁面與路由

| 路由路徑 | 頁面名稱 | 說明 |
|---------|---------|------|

### 2.2 互動功能

✅ **已包含功能**：
- 響應式導覽列（Desktop 固定置頂、Mobile 漢堡選單）
- 工程實績篩選（依類型、狀態即時篩選）
- 工程詳頁動態渲染（依 URL 參數載入資料）
- 圖片 Gallery 與 Lightbox
- 聯絡表單驗證與成功狀態
- 捲動式導覽列縮小效果
- 目前頁面高亮顯示
- 快速聯絡按鈕（電話、Email）

### 2.3 設計系統

- 完整的 Tailwind CSS 配置
- 元件化按鈕系統（Primary / Secondary / Accent）
- 卡片系統（Project / Service / News / Video）
- 表單元件（Input / Select / Textarea / Error / Success）
- Tag 標籤系統
- 響應式字級與間距規範
- 色彩 Token 系統

### 2.4 技術架構

- **前端框架**：Vite（多頁式應用）
- **CSS 框架**：Tailwind CSS 3.x
- **JavaScript**：原生 ES6+（無 jQuery 或大型框架依賴）
- **版本控制**：Git + GitHub
- **部署平台**：Vercel（支援自動部署）

## 3. 客戶需提供內容

所有實際內容需由客戶依照「附錄 A：內容提供清單」填寫，包含但不限於：

### 3.1 文案類

- 公司簡介、經營理念、品質政策
- 各服務項目說明文字
- 工程案例名稱、地點、說明
- 新聞標題與內容
- 職缺名稱與說明

### 3.2 圖片類

- 公司 Logo（SVG 或高解析度 PNG）
- 首頁 Hero 背景圖（建議尺寸：1920x1080px）
- 工程案例照片（每案至少 4 張，建議尺寸：1200x800px）
- 新聞/活動照片（建議尺寸：800x600px）
- 團隊照片（如適用）

### 3.3 資料類

- 工程案例完整資訊（名稱、地點、類型、規模、工期、業主、標籤、照片）
- 認證證書與獎項清單
- 職缺列表與職位需求
- 聯絡資訊（地址、電話、Email、營業時間）
- 社群媒體連結

### 3.4 檔案類

- 公司簡介 PDF
- 認證證書 PDF
- 其他可下載文件

## 4. 未包含項目

以下功能**不在本專案交付範圍內**，如需開發需另行估價：

❌ **後端與資料庫**
- 不提供後端 API 開發
- 不提供資料庫設計與部署
- 表單送出僅為前端驗證，不實際寫入資料庫

❌ **CMS 內容管理系統**
- 不提供後台管理介面
- 內容更新需透過修改 JSON 檔案或重新部署

❌ **會員系統**
- 員工專區僅為入口頁面，不含登入驗證功能
- 不提供會員註冊、登入、權限管理

❌ **進階互動功能**
- 不包含動畫特效（僅基本 hover / transition）
- 不包含影片播放器自訂（使用瀏覽器原生或 YouTube 嵌入）
- 不包含即時聊天系統

❌ **第三方整合**
- 不包含 Google Maps 實際嵌入（工程地圖為占位區）
- 不包含社群媒體 API 串接
- 不包含金流系統

❌ **多語系**
- 僅提供繁體中文版本

❌ **SEO 進階優化**
- 僅提供基本 meta tag 設定
- 不包含結構化資料（JSON-LD）
- 不包含 sitemap.xml 自動生成

## 5. 頁面規格

### 5.1 首頁 `/`

**頁面目的**：建立第一印象、引導訪客進入核心內容區

**區塊組成**：

| 區塊名稱 | 內容說明 | 互動功能 |
|---------|---------|---------|
| Hero 區 | 主標題、副標題、2 組 CTA 按鈕 | 按鈕導向聯絡頁、工程實績 |
| KPI 指標 | 4 組數字指標（例：完工案件數、經驗年數） | 無 |
| 核心能力 | 3 張服務卡片 | 點擊進入服務詳頁 |
| 代表工程 | 4-6 張工程卡片 | 點擊進入工程詳頁 |
| 工程地圖預覽 | 地圖視覺化示意圖 | 點擊進入完整地圖頁 |
| 品質肯定 | 認證 Logo 或獎項列表 | 點擊放大（Lightbox） |
| 新聞與影音 | 2-3 則最新消息 + 2 則影片 | 點擊進入詳頁或播放 |
| 最終 CTA | 大型聯絡按鈕與文案 | 導向聯絡頁 |

**RWD 行為**：
- **Desktop (≥1024px)**：雙欄或多欄排版，卡片並列
- **Tablet (768-1023px)**：單欄或雙欄，間距縮小
- **Mobile (<768px)**：單欄垂直排列，Hero 區移除長敘述，KPI 改為 2x2 網格

**空狀態處理**：
- 若代表工程資料不足 4 張：顯示「更多工程整理中」占位卡
- 若新聞資料為空：不顯示該區塊（整區隱藏）

**SEO 設定**：
- `<title>`：公司名稱 | 專業營造工程
- `<meta name="description">`：公司簡介摘要（150 字內）
- Open Graph 圖片：Hero 背景圖

**驗收標準**：
- [ ] Hero 區在各裝置下標題不破版
- [ ] KPI 數字正確顯示（使用實際提供數據）
- [ ] 所有 CTA 按鈕可點擊並導向正確頁面
- [ ] 卡片 hover 效果正常
- [ ] 手機版 Hero 不擠壓內容

## 6. 設計系統

### 6.1 色彩系統

**色彩 Token 定義**：

| Token 名稱 | 用途 | Tailwind 類別範例 |
|-----------|------|------------------|
| `color.primary` | 主色（導覽列、標題、主要按鈕） | `bg-brand-green` `text-brand-green` |
| `color.accent` | 強調色（CTA 按鈕、互動狀態） | `bg-brand-orange` `text-brand-orange` |
| `color.neutral.50-900` | 灰階（背景、次要文字） | `bg-slate-50` `text-slate-600` |
| `color.success` | 成功狀態 | `text-green-600` |
| `color.error` | 錯誤狀態 | `text-red-600` |

**色彩使用規則**：
- 主色用於品牌識別與內容引導
- 強調色僅用於 CTA 按鈕，不用於裝飾
- 避免同時使用主色與強調色在相鄰元素
- 文字與背景對比度需符合 WCAG AA 標準（4.5:1）

### 6.6 按鈕系統

**按鈕變體**：

| 變體名稱 | 視覺樣式 | 用途 |
|---------|---------|------|
| Primary | 主色填滿 + 白色文字 | 一般操作 |
| Secondary | 透明背景 + 主色邊框 + 主色文字 | 次要操作 |
| Accent | 強調色填滿 + 白色文字 | CTA 按鈕 |

**按鈕狀態**：
- Default：預設樣式
- Hover：稍微變深 + 陰影增強
- Active：更深 + 按下效果
- Disabled：降低不透明度 + 禁用互動

**按鈕尺寸**：
- Small：`px-3 py-1.5 text-sm`
- Medium：`px-4 py-2 text-base`（預設）
- Large：`px-6 py-3 text-lg`

**卡片變體**：

| 變體名稱 | 組成 | 用途 |

| News Card | 縮圖 + 標題 + 日期 + 摘要 | 新聞消息 |
| Video Card | 縮圖 + 標題 + 播放按鈕 | 影片 |

**卡片狀態**：
- Default：白色背景 + 輕微陰影
- Hover：陰影增強 + 稍微上移（`-translate-y-1`）

### 6.8 動效原則

**過渡時間**：
- 快速互動：150-200ms（按鈕、連結）
- 一般互動：300ms（卡片 hover、選單）
- 複雜動畫：500ms（Modal、抽屜）

**緩動函數**：
- 預設：`ease-in-out`
- 進入：`ease-out`
- 離開：`ease-in`

**動效使用規則**：
- 僅在有意義的互動上使用動效
- 避免過度動畫影響效能
- 尊重使用者的 `prefers-reduced-motion` 設定

## 11. 變更管理

### 11.1 變更範圍定義

**包含在原範圍內**（不另收費）：
- 文案內容修正（typo、用詞調整）
- 圖片替換（相同尺寸與版型）
- 色彩微調（主色色相偏移 ±10%）
- 既有功能的 Bug 修正

**超出原範圍**（需另行估價）：
- 新增頁面或區塊
- 修改版型結構（例如三欄改四欄）
- 新增互動功能（例如新增篩選條件）
- 整體設計風格大改
- 新增第三方服務整合

### 11.2 變更流程

1. **需求提出**：客戶以書面（Email）提出變更需求
2. **影響評估**：開發團隊評估工時與影響範圍（1-2 工作日）
3. **報價確認**：提供變更報價與時程影響
4. **客戶簽核**：客戶簽署變更單（Change Order）
5. **執行開發**：進入變更開發階段
6. **驗收確認**：客戶驗收變更項目

### 11.3 變更計價原則

| 變更類型 | 計價方式 |
|---------|---------|

## 15. 附錄 D：維護與更新指南

### 如何更新文案內容

1. 若為靜態文字：直接修改 HTML 檔案中的文字

3. 修改後執行 `npm run build` 並重新部署

### 如何新增工程案例

1. 打開 `src/data/projects.json`
2. 依照現有格式新增一筆資料
3. 將工程照片上傳至 `assets/` 資料夾
4. 重新建置與部署

### 如何修改品牌色彩

1. 打開 `tailwind.config.cjs`
2. 修改 `theme.extend.colors` 中的色碼
3. 執行 `npm run build`

### 如何新增頁面

1. 複製現有頁面 HTML 檔案
2. 修改內容
3. 在 `vite.config.js` 中新增入口點
4. 在 `src/components/layout.js` 中新增導覽連結（若需要）
5. 重新建置

### 如何部署更新

**自動部署（推薦）**：
1. 修改程式碼
2. `git add .`
3. `git commit -m "描述"`
4. `git push`
5. Vercel 自動部署（約 2 分鐘）

**手動部署**：
1. `npm run build`
2. 將 `dist/` 資料夾內容上傳至伺服器
